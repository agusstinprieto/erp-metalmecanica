What is McVill ERP? Introduction

McVill ERP is an enterprise resource planning system designed specifically for the metalworking and precision manufacturing industry. It integrates production, quality, human capital, sales, engineering, and financial analytics into a single platform powered by Artificial Intelligence.

Core Capabilities

  • +35 Integrated Modules — from high-level dashboards to industrial PPAP approvals
  • Gemini AI Integration in production, quality, automated quoting, visual inspection, and voice control
  • Multi-tenant Architecture — each company’s data is completely isolated and secure
  • Real-time Sychronization — Supabase Realtime updates system state instantly across devices
  • Hardware Ready — native support for barcode scanners, IP cameras, tablets, and label printers
  • Fully Responsive Web Portal — access from any device anywhere, no client installation required

Shop Floor Control

Complete lifecycle management of Work Orders (WO) from initial customer request to dispatch, tracked stage by stage.

Lifecycle of a Work Order

  1. WO is created with part number, customer ID, batch size, and target delivery date
  2. Process route is assigned: Cutting → Bending → Welding → Machining → Painting → Assembly
  3. Operator checks in: inputs employee ID to begin the assigned step
  4. Step completion: system logs actual processing time and forwards piece to the next stage
  5. AI visual inspection flags quality status at crucial nodes — auto-generates NCR on FAIL
  6. WO complete: updates finished goods inventory and closes the ledger

Delayed Status Detection Logic

remaining_hours = estimated_hours × (1 - progress_percentage/100)
days_to_finish = remaining_hours / 8
est_completion_date = today + days_to_finish

— Algorithm:
If est_completion_date > delivery_date → DELAYED
If remaining_days < 2 → AT RISK
Otherwise → ON TIME

Non-Conformance Report (NCR) Flow

  1. AI Vision flags FAIL → NCR is automatically generated with photo evidence attached
  2. System assigns severity: Critical (>85% confidence) · Major (>60%) · Minor (<60%)
  3. Quality supervisor receives an instant push notification on the dashboard
  4. Root Cause Analysis (RCA) and Corrective Action (CAPA) are logged
  5. Workflow tracking: Open → Investigating → Verifying → Closed
A Critical NCR immediately locks the active Traveler, preventing further processing until a supervisor bypasses or resolves the issue.

Clock-in Operational Flow

  1. Employee arrives at work — approaches the attendance terminal
  2. Badge Scan — operator places badge under the scanner; code `EMP-001` is captured
  3. Database Validation — system queries the profiles table in real time
  4. Auto Check-in — if no entry exists for the calendar date, it logs CHECK-IN with exact timestamp
  5. Visual Confirmation — terminal displays the employee's profile card in green
  6. End of Shift — employee scans badge again at exit
  7. Auto Check-out — logs exit timestamp, calculating total daily runtime minutes
  8. Payroll Sync — logs are committed to the secure ledger for payroll processing

Payroll & Formulas Reference Guide

The payroll processor uses timecard records from the attendance station to compute the net wages, including overtime, performance bonuses, and local tax deductions.

Base Variables

Variable KeyValue (Configurable)Description
DAILY_WAGE$450 MXNBase daily wage, configured individually per profile
OVERTIME_RATE× 2.0Standard multiplier for normal overtime hours
OEE_BONUS_THRESHOLD85%Target OEE required to unlock productivity bonuses
OEE_BONUS_PCT5%Productivity bonus percentage applied to gross salary
UMA Daily Index$108.57 MXNUnit of Measure and Indexation (Reference index)

Step 1 — Fetching Timecard Summaries

days_worked = COUNT(valid logs containing check_in AND check_out for the pay period)
overtime_hours = SUM(daily_overtime_minutes) / 60  → triggers only if daily OT > 30 min
days_absent = business_days - days_worked
days_late = COUNT(records with check_in > shift_start + 5 min)

Step 2 — Base Wage Calculation

base_wage = daily_wage × days_worked

Example: $450 × 22 days = $9,900

Step 3 — Overtime Payments

hourly_rate = daily_wage / 8
overtime_amount = overtime_hours × hourly_rate × 2.0

Example: 10 hrs OT × ($450/8) × 2 = $1,125

Step 4 — Productivity OEE Bonus

If oee_percentage ≥ 85% :
  bonus_oee = (base_wage + overtime_amount) × 0.05
If oee_percentage < 85% :
  bonus_oee = $0

Example: ($9,900 + $1,125) × 0.05 = $550.25

Step 5 — Absence Deductions

deductions_absence = days_absent × daily_wage

Example: 2 absences × $450 = $900 deduction

Step 6 — Gross Salary Sum

gross_salary = base_wage + overtime_amount + bonus_oee - deductions_absence

Example: $9,900 + $1,125 + $550 - $900 = $10,675 gross

Step 7 — Tax & Social Deductions

Applies local tax brackets and mandatory social security contributions (e.g. IMSS) on the calculated gross wage.

taxable_income = gross_salary - exempt_allowances
net_tax = calculated_tax_bracket - tax_subsidies
social_contribution = gross_salary × 0.01875  → Standard social security index rate

Step 8 — Net Payable Salary

total_deductions = net_tax + social_contribution + deductions_absence
net_salary = gross_salary - total_deductions

Example: $10,675 gross - $336.67 tax - $203.26 insurance = $10,135.07 NET

Control Limit Calculations

X̄̄ = Grand average of all subgroup averages
R̄ = Average range of all subgroups

UCL_X̄ = X̄̄ + A2 × R̄  ← Upper Control Limit for X̄
LCL_X̄ = X̄̄ − A2 × R̄  ← Lower Control Limit for X̄

UCL_R = D4 × R̄  ← Upper Control Limit for R
LCL_R = D3 × R̄  ← Lower Control Limit for R (0 if n ≤ 6)

σ = R̄ / d2  ← Process standard deviation

Process Capability Indexes

Cp = (USL − LSL) / (6 × σ)  ← Potential Capability
Cpk = min( (USL − X̄̄)/(3σ) , (X̄̄ − LSL)/(3σ) )  ← Actual Process Capability
